GitHub 가입 & CLI(gh) 설치

Git이 로컬 버전 관리 엔진이라면, GitHub는 그 엔진을 인터넷 위에 연결하는 클라우드 허브입니다. GitHub CLI(gh)는 브라우저 없이 터미널에서 저장소 생성·클론·코드 공유까지 처리하는 공식 도구입니다. 이 교안의 PART 1 스킬 클론, PART 3 knowledge-vault 클론 모두 gh로 처리할 수 있습니다.

Windows PowerShell — 네이티브 설치 (1줄)

 PowerShell · 네이티브 설치 (권장)
PS> irm https://claude.ai/install.ps1 | iex
⚠️ PowerShell PATH 문제 해결 — "claude를 찾을 수 없습니다" 오류 시
설치 후 claude 명령이 인식되지 않으면 아래 순서로 해결하세요.

① PATH에 설치 경로 수동 추가

 PowerShell PATH 등록
PS> $p = [Environment]::GetEnvironmentVariable('PATH', 'User')
PS> [Environment]::SetEnvironmentVariable('PATH', "$p;$env:USERPROFILE\.local\bin", 'User')

② 터미널 완전 종료 후 재시작 → 확인

Claude Desktop 앱 충돌 주의 — Claude Desktop 앱이 설치돼 있으면 WindowsApps\Claude.exe가 PATH를 선점해 CLI가 묻힐 수 있습니다. 이 경우 where claude 명령으로 어느 경로가 먼저 실행되는지 확인하고, %USERPROFILE%\.local\bin이 위에 오도록 PATH 순서를 조정하세요.

구조 인덱스 자동 재생성 스크립트 (update-structure.sh)

왜 만드나 — LLM이 매번 find/ls로 전체 탐색하면 토큰이 낭비됩니다. STRUCTURE.md 한 파일만 읽으면 현재 배치를 파악할 수 있게 합니다.

~/work/update-structure.sh 파일을 아래 내용으로 작성합니다.

 update-structure.sh 전체
$ cat > ~/work/update-structure.sh <<'EOF'
#!/usr/bin/env bash
# STRUCTURE.md 자동 재생성기 · 코드 폴더만 재귀 · 멱등
cd "$(dirname "$0")" || exit 1
OUT="STRUCTURE.md"; TODAY="$(date +%Y-%m-%d)"

walk() {
  find "$1" -mindepth 1 -maxdepth 1 -type d | sort | while read -r d; do
    n=$(basename "$d"); code="${n%%-*}"
    echo "$2$n/"
    [[ "$code" =~ ^[0-9]+$ ]] && walk "$d" "$2  "
  done
}

gen() {
  echo "# STRUCTURE.md — 구조 인덱스 (자동 생성)"
  echo; echo "최종 자동 갱신: $TODAY"; echo
  echo '```'
  for cat in 0-parking 1-process 2-routine; do
    [ -d "$cat" ] || continue
    echo "$cat/"; walk "$cat" "  "; echo
  done
  echo '```'
}

new="$(gen)"
printf '%s\n' "$new" > "$OUT"
echo "✓ STRUCTURE.md 갱신 ($TODAY)"
EOF
$ chmod +x ~/work/update-structure.sh

📍 현재 OS : 🪟 Windows — 아래 명령어 2개를 실행하세요. 🍎 macOS — Mac 전용 설정이 추가되어 3개를 실행하세요. | 다른 OS라면 →

Mac 필수 설정 3개 — Mac은 NFC/NFD 유니코드 방식 차이로 Windows보다 설정이 하나 더 필요합니다. 모두 실행하세요.

 터미널 · 명령어 2 / 3 MAC 
$ git config --global core.precomposeunicode true

③ 대소문자 구분 강제 적용 — macOS 기본값(대소문자 무시)이 서버·Windows와 충돌하는 현상 방지

$ git config --global core.ignorecase false
--global 의미 — 이 PC의 모든 Git 프로젝트에 한 번만 설정하면 영구 적용됩니다. 다시 입력할 필요 없습니다.

무작위로 깃을 설정하면 PC가 멈추거나 통합 검색이 불가능해집니다. 디스크 상황에 맞춰 유형 A · B 중 하나를 골라 세팅하세요.

STEP 3-1 · 유형 A

폴더 만들고 에디터로 열기

  1. D드라이브에 프로젝트 폴더 생성 (예: D:\Total_Projects\A아파트_설계공사)
  2. 그 안에 캐드(.dwg)·엑셀(.xlsx)·워드(.docx) 실무 파일을 넣기
  3. 에디터에서 File > Open Folder...로 해당 폴더 열기
  1. Finder에서 프로젝트 폴더 생성 (예: ~/Documents/Total_Projects/A아파트_설계공사)
  2. 그 안에 캐드(.dwg)·엑셀(.xlsx)·워드(.docx) 실무 파일을 넣기
  3. 에디터에서 File > Open Folder...로 해당 폴더 열기
STEP 3-2 · 유형 A

로컬 저장소 초기화(init)

👤 직원 → 🤖 AI 직원 지시

이 폴더를 로컬 Git 저장소로 초기화(init)해 줘. 인터넷 연결 없이 내 컴퓨터에서만 로컬로 쓸 거야.

STEP 3-1 · 유형 B

작업 폴더 / 기록 창고 분리 준비

  1. C드라이브(SSD)에 실제 작업 폴더 생성 (예: C:\Working\A아파트_설계공사)
  2. D드라이브(HDD)·외장하드에 기록 보관용 빈 폴더 생성 (예: D:\Git_Archive\A아파트_설계공사)
  3. 에디터로 C드라이브 작업 폴더를 Open Folder
  1. 빠른 드라이브에 실제 작업 폴더 생성 (예: ~/Documents/Working/A아파트_설계공사)
  2. 외장하드 또는 별도 볼륨에 기록 보관용 빈 폴더 생성 (예: /Volumes/Archive/Git_Archive/A아파트_설계공사)
  3. 에디터로 작업 폴더(~/Documents)를 Open Folder
STEP 3-2 · 유형 B

기록 창고를 외부 경로로 분리 개설

👤 직원 → 🤖 AI 직원 지시 WIN

현재 내 작업 폴더의 Git 기록 창고(.git)를 내가 지정한 외부 경로인 D:\Git_Archive\A아파트_설계공사 폴더에 별도로 분리해서 개설해 줘.

👤 직원 → 🤖 AI 직원 지시 MAC

현재 내 작업 폴더의 Git 기록 창고(.git)를 내가 지정한 외부 경로인 /Volumes/Archive/Git_Archive/A아파트_설계공사 폴더에 별도로 분리해서 개설해 줘.

결과 — 무거운 과거 데이터는 D드라이브에 쌓이고, C드라이브엔 연결용 이정표 파일(1KB)만 남아 SSD 용량이 보존됩니다.

❌ 기존 방식

STEP 7-3 · 핵심 원칙

원본 무결성 — 1차 지식과 AI 합성 지식을 섞지 않는다

왜 중요한가 — AI가 기존 지식 노트들을 연결해 만든 답변을 그대로 저장하면, 원본과 2차 합성물이 뒤섞여 정보가 왜곡됩니다. 이를 막기 위해 모든 노트에 tier를 붙입니다.
📊 tier 구분 — 지식의 신뢰도 계층
primary
프로젝트에서 직접 추출한 1차 지식 — source:로 원천 프로젝트를 링크.
예) A아파트 공사에서 뽑아낸 "로비 계단 설계 교훈"
synthesized
여러 지식 노트를 AI로 합성한 2차 지식 — derived_from:에 출처 노트 목록, status: draft 표시.
원본과 동급 권위 아님 — 검증 전엔 보조 참고로만 사용.
승격 조건 — synthesized 노트를 primary로 격상하려면, 다른 지식 노트가 아닌 실제 원천 프로젝트와 재대조해 검증 후 /promote 커맨드를 사용합니다. AI 합성물이 파생물을 낳는 드리프트를 원천 차단합니다.
